🗺️ batimetria_kml_shape

Suite de conversores Python para transformar levantamentos batimétricos (CSV, TXT, XYZ, Excel, LOG) em shapefiles prontos para GIS — contornos de área, linhas de navegação e exportação em SIRGAS 2000 UTM.

Repositório local SIRGAS 2000 UTM tkinter GUI

Problema que resolve

Levantamentos batimétricos chegam em planilhas, arquivos de texto e logs heterogêneos. Antes de usar no QGIS ou ArcGIS, é preciso gerar geometrias confiáveis — sem juntar áreas desconectadas, sem duplicar arquivos idênticos e com filtros de qualidade por profundidade e coordenadas.

Ferramentas incluídas

  • converter_shapefile_batimetria.py v1.8.0 · Contornos batimétricos Gera polígonos de área (CA_<arquivo>_<pasta>.shp) e linhas de navegação (L_...). Detecta áreas espacialmente desconectadas, deduplica conteúdo equivalente e aplica envoltória côncava por área.
  • converter_linhas_batimetria.py v1.0.1 · Linhas batimétricas Exporta apenas linhas de navegação (LineString) — um shapefile por arquivo de origem, sem polígonos de área.
  • converter_batimetria2.py v1.6.0 · Motor de leitura Biblioteca compartilhada: leitura multi-formato, filtros geográficos, agrupamento por trilha, fingerprint para deduplicação e conversão para KML.

Inteligência implementada

Áreas desconectadas

Grade UTM adaptativa separa múltiplas áreas no mesmo arquivo (ex.: Leplac Amapá).

Deduplicação

Fingerprint normalizado ignora arquivos com conteúdo batimétrico equivalente.

Trilhas OBS

Agrupamento por trilha de observação gera polígonos/linhas por passagem.

Filtros de qualidade

Outliers, profundidade inválida e pontos fora do Brasil são descartados.

Como executar

cd batimetria_kml_shape
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

# Contornos + linhas (diálogo de pasta)
python converter_shapefile_batimetria.py

# Apenas linhas de navegação
python converter_linhas_batimetria.py

Entrada: pasta com CSV, TXT, XYZ, XLS, XLSX ou LOG. Saída: shapefiles CA_* e L_* na mesma pasta selecionada.

Formatos de entrada

  • Planilhas Excel (multi-aba, com suporte a GPS embarcado em células)
  • Arquivos XYZ e CSV delimitados
  • Logs HYPACK e arquivos de texto batimétrico
  • Coordenadas WGS84 ou UTM — reprojeção automática para SIRGAS 2000

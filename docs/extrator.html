📋 extrator_info_files

Inventário inicial de campanha em minutos, não em dias — descubra o que há na pasta, de que tipo é cada dado geofísico e se existe posição no mapa, sem abrir arquivo por arquivo.

v0.2.1 Open source 100% local

Antes e depois

Antes (manual)

Abrir cada arquivo no software de processamento; horas ou dias para inventariar um HD.

Depois (extrator)

Uma varredura na pasta inteira; tabela com tipo, estágio e coordenadas em minutos.

Para quem é

  • Operacional / processamento — saber o que importar primeiro ao receber um HD de campanha
  • QA / entrega / gestão — conferir tipos esperados, GPS presente, pares .all/.wcd
  • TI / geoprocessamento — código modular, amostras sintéticas e testes automatizados

Formatos suportados (v1)

  • HYPACK (.raw, .hsx)
  • Kongsberg EM (.all, .wcd)
  • EdgeTech XTF / JSF (.xtf, .jsf)
  • CARIS / SES (.raw SFD)
  • Surfer grids (.rsw, .grd)
  • E classificação genérica para extensões desconhecidas

Demo em 30 segundos

git clone https://github.com/AndressaLF/extrator_info_files.git
cd extrator_info_files
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -e .
python main.py banco_de_dados_treinamento/amostras_sinteticas -r

Para sua pasta de campanha: python main.py "D:\SuaPasta" -r --pdf

Exportações

  • Tabela colorida no terminal (-r)
  • Relatório PDF formal (--pdf)
  • CSV e JSON estruturado com schema_version para pipelines
  • Modo strict para QA automatizado (--strict)

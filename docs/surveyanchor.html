⚓ SurveyAnchor

Ancore levantamentos de campo à sua localização geográfica. Lê planilhas, textos, logs e outros formatos batimétricos, reconhece automaticamente o layout e o sistema de coordenadas, e exporta um KML enxuto (e shapefiles opcionais) para consultar no Google Earth — sem software GIS proprietário.

Status atual: v0.7.3 (alpha em calibração). Processamento 100% local.

📍 Geolocalização batimétrica 🗺️ KML / SHP 🔒 100% local 📚 Docs online

🎯 O que ela resolve

📦 A situação comum

Acervos batimétricos chegam misturados (Excel, TXT, LOG, NMEA, BAG…) sem documentação clara de coluna, CRS ou área. A pergunta básica fica: onde exatamente aconteceu este levantamento?

✨ Com o SurveyAnchor

O motor identifica formato e coordenadas, limpa outliers e entrega posição geográfica consultável — KML amostrado (~2.500 pontos) e, se quiser, contornos e linhas em shapefile.

🧰 Comandos principais (CLI)

  • 🎯 surveyanchor export-kml KML enxuto Abre seletor de pasta e gera *.kml com pontos amostrados e profundidade — ideal para confirmar a área no Google Earth.
  • 📐 surveyanchor export-shp-contours Contornos + linhas Shapefiles CA_*.shp (área) e L_*.shp (navegação).
  • 〰️ surveyanchor export-shp-lines Só linhas Exporta apenas linhas de navegação (LineString).
  • 📋 surveyanchor catalog -i PASTA Catálogo Inventário tabular (CSV / JSON / SQLite) dos arquivos reconhecidos.
  • 📁 surveyanchor organize -i PASTA Organizar acervo Dry-run por padrão; com --execute reorganiza cópias por ano.

📂 Formatos (resumo)

Texto tabular, NMEA, Excel multi-aba, grades BAG, nuvens LAS, rasters GeoTIFF e parsers leves de multibeam (GSF, XTF, KMALL), entre outros — em calibração contínua.

  • 📊 Planilhas e TXT/XYZ/CSV com coordenadas + profundidade
  • 📡 Logs NMEA / ecosonda
  • 🗺️ BAG, GeoTIFF, LAS e formatos de campo adicionais

▶️ Como começar

cd surveyanchor
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -e .

surveyanchor export-kml -r
surveyanchor catalog -i "D:\Levantamentos" -o "D:\Saida\catalogo" -r
